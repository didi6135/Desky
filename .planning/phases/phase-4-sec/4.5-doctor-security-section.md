## Goal

Add a "Security" section to `doctor.sh` that verifies the hardening
hasn't drifted. Six months from now, an upgrade or manual edit
shouldn't be able to silently revert our hardening — doctor catches
it.

## Why

Hardening is one-shot work; drift is forever. Without an automated
check, the only way to know the unit is still hardened is by
periodically running `systemd-analyze` by hand. Doctor is where
this naturally lives.

## The new section

Insert in `doctor.sh` after the existing "Manifests" section,
before "Claude Code state":

```bash
# ─── Security ─────────────────────────────────────────────────────────────
section "Security  (hardening + permissions drift check)"

# 1. systemd-analyze score
if command -v systemd-analyze >/dev/null 2>&1; then
  score=$(systemd-analyze --user security claude-telegram.service 2>/dev/null \
    | awk '/Overall exposure level/ {print $4}' || true)
  if [[ -n "$score" ]]; then
    # Compare floats: bash arithmetic is integer-only, so multiply by 10
    score_x10=$(echo "$score * 10" | bc 2>/dev/null | cut -d. -f1)
    if [[ -n "$score_x10" ]] && [[ "$score_x10" -le 30 ]]; then
      check "systemd-analyze exposure: $score (≤ 3.0)" 0
    elif [[ -n "$score_x10" ]] && [[ "$score_x10" -le 50 ]]; then
      check "systemd-analyze exposure: $score (target: ≤ 3.0)" 2 \
        "Some Tier-2/3 hardening may have drifted. Re-run install.sh."
    else
      check "systemd-analyze exposure: $score (UNSAFE — target: ≤ 3.0)" 1 \
        "Hardening reverted. Re-run install.sh to restore the unit."
    fi
  else
    check "systemd-analyze score (unparseable)" 2 \
      "systemd-analyze didn't return a score. Manual check: systemd-analyze --user security claude-telegram"
  fi
else
  check "systemd-analyze not available" 2 \
    "Install systemd-analyze for full security checks. Otherwise skip."
fi

# 2. Tier-1 directives present in unit
if [[ -s "$SERVICE_UNIT" ]]; then
  required=(
    "NoNewPrivileges=true"
    "PrivateTmp=true"
    "ProtectKernelTunables=true"
    "ProtectKernelModules=true"
    "ProtectControlGroups=true"
    "RestrictNamespaces=true"
    "MemoryMax="
    "TasksMax="
  )
  missing=()
  for d in "${required[@]}"; do
    grep -qE "^${d}" "$SERVICE_UNIT" || missing+=("$d")
  done
  if [[ ${#missing[@]} -eq 0 ]]; then
    check "Tier-1 hardening directives present" 0
  else
    check "Tier-1 hardening: ${#missing[@]} directive(s) missing" 1 \
      "Missing: ${missing[*]}" \
      "Re-run install.sh to restore the hardened unit."
  fi
fi

# 3. Filesystem write-restriction
if grep -qE "^ProtectHome=read-only" "$SERVICE_UNIT" 2>/dev/null; then
  check "ProtectHome=read-only" 0
else
  check "ProtectHome=read-only missing" 1 \
    "Bot can write anywhere in \$HOME. Re-run install.sh."
fi

if grep -qE "^ReadWritePaths=.*\.claudify" "$SERVICE_UNIT" 2>/dev/null; then
  check "ReadWritePaths includes ~/.claudify" 0
else
  check "ReadWritePaths missing or incomplete" 1 \
    "ProtectHome may be read-only without an exception for ~/.claudify. Re-run install.sh."
fi

# 4. Address-family + syscall filter (3.6.3)
if grep -qE "^RestrictAddressFamilies=" "$SERVICE_UNIT" 2>/dev/null; then
  check "RestrictAddressFamilies set" 0
else
  check "RestrictAddressFamilies missing" 2 \
    "Tier-3 hardening not applied. systemd-analyze score will be higher than target."
fi

if grep -qE "^SystemCallFilter=" "$SERVICE_UNIT" 2>/dev/null; then
  check "SystemCallFilter set" 0
else
  check "SystemCallFilter missing" 2 \
    "Tier-3 hardening not applied."
fi

# 5. Permissions on sensitive files
for f in "$CREDS_FILE" "$CLAUDIFY_TELEGRAM/.env" "$CLAUDIFY_TELEGRAM/access.json"; do
  if [[ -s "$f" ]]; then
    perms=$(stat -c '%a' "$f")
    if [[ "$perms" == "600" ]]; then
      check "$(basename "$f") mode 600" 0
    else
      check "$(basename "$f") mode $perms (expected 600)" 1 \
        "chmod 600 $f"
    fi
  fi
done

# 6. No tokens in process command line
if pgrep -af 'claude' | grep -qE 'sk-ant-|TELEGRAM_BOT_TOKEN='; then
  check "Tokens may be visible in ps output" 1 \
    "A token leaked into a process arg. Investigate immediately."
else
  check "No tokens in process command lines" 0
fi

# 7. credentials.env owned by current user
if [[ -s "$CREDS_FILE" ]]; then
  owner=$(stat -c '%U' "$CREDS_FILE")
  if [[ "$owner" == "$USER" ]]; then
    check "credentials.env owned by $USER" 0
  else
    check "credentials.env owned by $owner (expected $USER)" 1 \
      "chown $USER $CREDS_FILE"
  fi
fi
```

The check count goes from 30 to ~38, depending on how many tier-3
checks ship.

## Notes / risks

- **`systemd-analyze`'s output format may shift between versions.**
  The awk parse for `Overall exposure level` is brittle; a better
  long-term approach is `systemd-analyze --user security
  claude-telegram --json=short` and parse JSON. But that's a
  newer-systemd feature; awk is more portable. Document the
  fallback.
- **bc may not be installed.** It's part of POSIX userspace but
  some minimal containers omit it. Fall back to integer comparison
  via `${score%.*}` if `bc` is missing.
- **The check count drift** when these checks land — the doctor
  banner says "All N checks passed" with a dynamic N, so adding
  checks doesn't break the existing assertion. But the README +
  install summary may quote "28 checks" — update those too.
