## Goal

After install, the operator runs `<name> doctor`, `<name> update`,
`<name> status`, etc. — short, personal commands instead of long
`claudify --name=<name> doctor` invocations. The wrapper script
lives at `~/.local/bin/<name>` and dispatches to the right entrypoint
+ systemctl call.

## Why

`docs/architecture.md §3d` describes this as a core UX improvement
for multi-instance users. *"david doctor"* is dramatically more
ergonomic than *"claudify --name david doctor"*. Once an instance
exists, the personal command becomes the daily interface.

## The wrapper script (generated)

```bash
#!/usr/bin/env bash
# Auto-generated by Claudify install for instance "<NAME>"
# Edit at ~/.local/bin/<NAME> only if you know what you're doing —
# this file gets regenerated on every Claudify update.

CLAUDIFY_INSTANCE="<NAME>"

case "${1:-}" in
  doctor)    exec bash <(curl -fsSL https://raw.githubusercontent.com/didi6135/Claudify/main/doctor.sh) --name "$CLAUDIFY_INSTANCE" ;;
  update)    exec bash <(curl -fsSL https://raw.githubusercontent.com/didi6135/Claudify/main/update.sh) --name "$CLAUDIFY_INSTANCE" ;;
  uninstall) exec bash <(curl -fsSL https://raw.githubusercontent.com/didi6135/Claudify/main/uninstall.sh) --name "$CLAUDIFY_INSTANCE" "${@:2}" ;;
  status)    systemctl --user status  "claudify-$CLAUDIFY_INSTANCE" ;;
  logs)      journalctl --user -u     "claudify-$CLAUDIFY_INSTANCE" -f ;;
  restart)   systemctl --user restart "claudify-$CLAUDIFY_INSTANCE" ;;
  start)     systemctl --user start   "claudify-$CLAUDIFY_INSTANCE" ;;
  stop)      systemctl --user stop    "claudify-$CLAUDIFY_INSTANCE" ;;
  ""|help|-h|--help)
    cat <<HELP
$CLAUDIFY_INSTANCE — your Claudify instance "${CLAUDIFY_INSTANCE}"

Usage:
  $CLAUDIFY_INSTANCE doctor      Run health checks
  $CLAUDIFY_INSTANCE update      Pull the latest Claudify and refresh in place
  $CLAUDIFY_INSTANCE uninstall   Remove this instance entirely
  $CLAUDIFY_INSTANCE status      Show systemd service status
  $CLAUDIFY_INSTANCE logs        Follow service logs
  $CLAUDIFY_INSTANCE restart     Restart the service
  $CLAUDIFY_INSTANCE start       Start the service
  $CLAUDIFY_INSTANCE stop        Stop the service
HELP
    ;;
  *)
    echo "Unknown subcommand: $1" >&2
    "$0" --help
    exit 2
    ;;
esac
```

## `lib/personal-cmd.sh` — public functions

| Function | Args | Behavior |
|---|---|---|
| `personal_cmd_install` | `<name>` | Generate wrapper at `~/.local/bin/<name>`, chmod 755, ensure PATH |
| `personal_cmd_uninstall` | `<name>` | Remove wrapper file (idempotent) |
| `personal_cmd_ensure_path` | — | Add `~/.local/bin` to PATH in `~/.bashrc` and `~/.zshrc` if not present |

## Steps

1. Write `lib/personal-cmd.sh` with the 3 functions
2. Add to `build.sh` MODULES; source order in `install.sh` after `manifest.sh`
3. `install.sh` main(): after `manifest_register_instance`, call `personal_cmd_install $INSTANCE_NAME`
4. `uninstall.sh`: call `personal_cmd_uninstall $INSTANCE_NAME` before removing the instance dir
5. PATH update: append `export PATH="$HOME/.local/bin:$PATH"` to `~/.bashrc` AND `~/.zshrc` (whichever exist) if and only if `~/.local/bin` isn't already in `$PATH`. Print a notice if the user's shell needs to source the rc file or open a new terminal for `<name>` to work.
6. Final summary in `lib/service.sh` prints something like:
   ```
   Run any of these:
     <name> status        — service health
     <name> logs          — follow logs
     <name> doctor        — full diagnostic
     <name> --help        — full list
   ```
   (replacing the systemctl one-liners)
7. `tests/bash/personal-cmd.bats`:
   - Generate wrapper for fake instance `test-x`, assert exists + is executable
   - Wrapper's `--help` output mentions `test-x`
   - PATH update is idempotent (running install twice doesn't double the line)
8. **Station11 round-trip:**
   - `bash ~/test/uninstall.sh --name default --yes`
   - `bash ~/test/autoinstall.sh`
   - `which default` — finds wrapper
   - `default --help` — prints help
   - `default status` — shows service active
   - `default logs` (Ctrl-C) — works
9. CHANGELOG: `### Added — personal command wrapper at ~/.local/bin/<name>. After install, run "<name> doctor", "<name> update", "<name> status", etc. directly.`

## Notes / risks

- PATH manipulation in rc files is sensitive. Test idempotency hard. Use a marker comment so we can clean up reliably on uninstall (e.g., `# Claudify PATH —`).
- Some shells (sh, dash) won't read `.bashrc` — fine, those users will need to fix their PATH manually. Document the limitation in the post-install summary.
- Validate the instance name doesn't collide with a system command BEFORE generating the wrapper. `validate_instance_name` (added in 3.4.5) handles this; double-check it's called.
