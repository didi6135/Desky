## What landed

A wrapper at `~/.local/bin/<NAME>` that dispatches per-instance
subcommands. After install the operator runs `<name> doctor` instead
of `bash uninstall.sh --name <name>` — short, ergonomic per-instance
control surface (docs/architecture.md §3d).

Per ADR 0006 the wrapper is engine-agnostic: nothing inside it
references `claude` or any engine adapter; it only deals with
systemctl + Claudify entrypoints.

## Design choices

| Decision | Why |
|---|---|
| **Wrapper uses `bash <(curl …)` for doctor/update/uninstall** | Matches today's `update.sh` model — the operator's command always pulls today's entrypoint from origin, so a stale wrapper doesn't lock them onto an old `doctor`/`uninstall`. CLAUDE.md's "no `bash <(curl …)` without explaining" rule is about *us* fetching code mid-session; here the operator chose to run `<name> update` themselves. |
| **systemctl/journalctl subcommands stay local** | `status / logs / start / stop / restart` need no network — direct `systemctl --user … claudify-<name>` keeps them instant and offline-safe. |
| **Marker-tagged PATH line (`# Claudify PATH —`)** | One unambiguous string makes uninstall's awk removal reliable, immune to operator-added PATH lines, and survives `personal_cmd_ensure_path` running twice. |
| **`personal_cmd_ensure_path` updates both `~/.bashrc` AND `~/.zshrc` when present** | Operator's daily shell isn't ours to assume. Bash + Zsh cover ~99% of Ubuntu/Debian operators; `sh` / `dash` users handle PATH manually (documented in the spec's "Notes / risks"). |
| **`personal_cmd_clean_path` only fires when the registry empties** | Matches the existing registry-file-cleanup pattern in uninstall.sh: shared state survives partial uninstalls, only the *last* removal wipes shared state. |
| **Wrapper file regenerated on every install (overwrite, not patch)** | Updates ship a new template (e.g., a new subcommand) without diffing logic — last install always wins, and the operator can't end up with a half-old wrapper that lacks subcommands the spec promises. |
| **Idempotency baked into every public function** | `install` overwrites cleanly; `uninstall` handles already-gone files; `ensure_path` greps-then-appends; `clean_path` no-ops without the marker. Matches CLAUDE.md rule 8. |
| **No `~/.local/bin/<name>` doctor fallback for offline installs** | If the operator has no network, `<name> doctor` will fail — but so will everything else Claudify needs (Telegram API, raw.githubusercontent for update, the bot's polling). Building offline-doctor branching is yak-shaving for a network-required product. |
