<rename_rule>
**The exact mechanical operation. Replace these three case-distinct tokens:**
- `CLAUDIFY` → `DESKY`   (constants / env-var prefixes, e.g. `CLAUDIFY_INSTANCE_DIR`)
- `Claudify` → `Desky`   (brand strings in banners, comments, help text)
- `claudify` → `desky`   (paths, service/registry/manifest names, markers, log files)

This token set never overlaps with `claude`/`Claude`/`CLAUDE` (they diverge at the
6th character: `claud`**`i`**`fy` vs `claud`**`e`**), so the replace is safe by
construction.

**PRESERVE (engine identity — must remain byte-for-byte):**
`claude-code` (engine id + `lib/engines/claude-code.sh` filename + `build.sh`
MODULES entry), `CLAUDE_CONFIG_DIR`, `.claude.json`, `CLAUDE.md` (persona
filename), `claude mcp`, the `claude` binary, and the product name "Claude Code".

**TWO EXCLUSIONS (revert after the global pass):**
1. **GitHub URL values** — the substring `didi6135/Claudify` inside
   `github.com/...` and `raw.githubusercontent.com/...` URLs must stay
   `didi6135/Claudify` in this plan (the repo is not renamed until 01-02; keeping
   it means curl URLs still resolve). The *variable* `CLAUDIFY_RAW_BASE` is renamed
   to `DESKY_RAW_BASE`, but its URL *value* keeps `didi6135/Claudify`.
2. **Bead issue IDs** — `Claudify-e4a` (8 occurrences across `lib/args.sh`,
   `lib/personal-cmd.sh`, `lib/validate.sh`, `tests/bash/multi-instance.bats`,
   `tests/bash/personal-cmd.bats`) are immutable tracker identifiers (bead prefix
   is literally "Claudify"). Leave them as `Claudify-e4a`.

**Recommended execution:** run the three-casing replace globally over the in-scope
files, then restore the two exclusions:
`Desky-e4a` → `Claudify-e4a` and the URL substring `didi6135/Desky` →
`didi6135/Claudify`. Then grep to prove only the intended exceptions remain.
</rename_rule>
