### `engine_apply_persona <text>`

**Args:** the rendered persona text (a small markdown block).
**Idempotent.**

```bash
engine_apply_persona() {
  local rendered="$1"
  local target="$CLAUDIFY_INSTANCE_DIR/workspace/CLAUDE.md"
  local marker_start='<!-- claudify:persona:start -->'
  local marker_end='<!-- claudify:persona:end -->'
  local block="$(printf '%s\n%s\n%s\n' "$marker_start" "$rendered" "$marker_end")"

  if [[ -s "$target" ]] && grep -q "$marker_start" "$target"; then
    # Replace the existing block in place; awk preserves rest of file.
    awk -v start="$marker_start" -v end="$marker_end" -v new="$block" '
      $0 ~ start { print new; in_block=1; next }
      in_block && $0 ~ end { in_block=0; next }
      in_block { next }
      { print }
    ' "$target" > "$target.tmp" && mv "$target.tmp" "$target"
  else
    # First call: prepend the block above any existing content.
    if [[ -s "$target" ]]; then
      printf '%s\n\n%s\n' "$block" "$(cat "$target")" > "$target.tmp"
      mv "$target.tmp" "$target"
    else
      printf '%s\n' "$block" > "$target"
    fi
  fi
  chmod 644 "$target"
}
```

The marker pair is what makes this safe to call repeatedly without
clobbering operator edits. Three behaviours, all tested:

| Scenario | Behaviour |
|---|---|
| First call on missing file | Writes block; file is just the block. |
| First call on existing CLAUDE.md (no markers) | Prepends block above existing content; markers + persona become the *first* thing the model sees. |
| Re-call with new text | Replaces only the bracketed region; everything outside the markers (including operator hand-edits) survives byte-for-byte. |
| Re-call with same text | File is byte-identical (verified via sha256 in smoke test). |

## Design choices

| Decision | Why |
|---|---|
| **`engine_memory_setup` ships as a no-op stub** | Adding a contract function before its body exists lets every callsite be written normally (no `[[ -n "$(declare -F engine_memory_setup)" ]] && engine_memory_setup` guards). When Phase 4.0b adds the real body, no other file changes. |
| **Marker convention `<!-- claudify:persona:start -->` / `:end -->`** | HTML comments render as nothing in markdown, so the markers don't pollute the persona text the model sees. They're discoverable by name (grep `claudify:persona`) and unambiguous (no operator is going to type those). |
| **awk replace, not sed** | sed's `c` command for "replace block between markers" is ugly across line boundaries. awk's "set in_block flag, skip until end marker, then print the new block once" is straightforward and portable across BSD/GNU. |
| **First-call behaviour: prepend, not append** | An operator with hand-written CLAUDE.md content keeps it intact; the persona block becomes the *first* thing the model reads. Putting persona last would mean the operator's content takes precedence — opposite of what we want. |
| **Engine-agnostic split: `engine_memory_setup` (MCP) + `engine_apply_persona` (context surface)** | Two surfaces, two functions. A future Gemini adapter might use Gemini's "system prompt" mechanism for persona and a different MCP-equivalent for tools — separating them lets each adapter implement what's natural for its engine. |
| **Per-engine subdir for the persona target — not `~/.claude/CLAUDE.md`** | The persona is per-instance (different bots can have different personas); Claude Code's per-instance config dir already routes `CLAUDE_CONFIG_DIR` correctly, so writing under `${CLAUDIFY_INSTANCE_DIR}/workspace/CLAUDE.md` keeps everything under the per-instance umbrella. |
