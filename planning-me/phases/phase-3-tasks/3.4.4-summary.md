## What landed

Two JSON files are now Claudify's source of truth for *what's
installed*: a top-level **registry** and a **per-instance manifest**.
Every entrypoint reads them; every install writes them.

## The two files

### `~/.claudify/instances.json` — the registry
```json
{
  "version": 1,
  "instances": {
    "default": {
      "created_at": "2026-05-04T13:17:40Z",
      "engine": "claude-code",
      "service": "claude-telegram",
      "personal_cmd": "default"
    }
  }
}
```
One entry per Claudify instance on this host. `doctor.sh` (multi-
instance) iterates this list. `update.sh` defaults to "all instances".
`uninstall.sh` requires explicit `--name <one>` or `--all`.

### `~/.claudify/claudify.json` — per-instance details
```json
{
  "version": 1,
  "name": "default",
  "created_at": "2026-05-04T13:17:41Z",
  "claudify_version": "0.1.0-dev",
  "engine": "claude-code",
  "engine_version": "2.1.126 (Claude Code)",
  "channels": {
    "telegram": {
      "installed_at": "2026-05-04T13:17:41Z",
      "enabled": true,
      "version": ""
    }
  },
  "mcps": {},
  "skills": [],
  "hooks": []
}
```
What `backup.sh` reads to know what to bundle. What `doctor.sh` reads
to know what to check. What `update.sh` reads to know what's enabled.
3.4.5 will move this file under `~/.claudify/instances/<name>/`.

## The contract — 10 functions

| Function | Args | Behaviour |
|---|---|---|
| `manifest_init_registry` | — | Create `instances.json` if missing. Idempotent. |
| `manifest_register_instance` | `<name>` | Add or refresh registry entry. **Preserves `created_at`** on re-register so the install timestamp survives `update.sh` runs. |
| `manifest_unregister_instance` | `<name>` | Remove a registry entry (used by future `uninstall.sh --name`). |
| `manifest_list_instances` | — | Echo each instance name on its own line. |
| `manifest_get_instance` | `<name>` | Print one registry entry as JSON (or `null` if absent). |
| `manifest_init_instance` | `<name>` | Create per-instance `claudify.json` if missing; on re-call, refreshes `claudify_version` + `engine_version` while preserving channels/mcps/skills/hooks. |
| `manifest_set_channel` | `<name> <channel> [version]` | Add/update a channel entry with `installed_at` + `enabled: true` + version. Auto-creates the per-instance manifest if missing. |
| `manifest_set_mcp` | `<name> <mcp> [version]` | Same shape, for MCPs. |
| `manifest_read_field` | `<name> <jq-path>` | Read one field via `jq -r`. Returns 1 if file missing. |
| `manifest_atomic_write` | `<file> <contents>` | Internal helper, exposed for tests. Writes `${file}.tmp`, then `mv`. |

## Design choices

| Decision | Why |
|---|---|
| **Path resolution centralised in `_registry_path` / `_instance_manifest_path`** | 3.4.5 moves the per-instance file under `instances/<name>/`. With path resolution in two helper functions, the migration is two lines of edits, not a hunt across callsites. |
| **`created_at` preserved on re-register** | Update flow (`update.sh` → `--preserve-state`) re-runs `manifest_register_instance default`. The created_at timestamp belongs to *the install*, not *the most recent boot* — preserving it across re-registers is what makes "when did this bot first come up?" answerable forever. |
| **Atomic writes via `.tmp` + `mv`** | `mv` is atomic on POSIX. Ctrl-C between `printf > tmp` and `mv` leaves the original manifest intact (the .tmp lingers but is harmless). The worst case is "stale .tmp" not "corrupt manifest", which is the only acceptable failure mode for a source-of-truth file. |
| **`manifest_set_channel` auto-creates the per-instance manifest** | One fewer ordering rule for callers. If you set a channel and the manifest doesn't exist yet, you get a fresh manifest with that channel — no "you forgot to call init_instance first" footgun. |
| **Empty version string acceptable for telegram** | We don't yet parse the actual telegram plugin version from `claude plugin list`. Empty string keeps the field present (schema-compliant) without faking a value. 3.4.4 follow-up could parse the real version. |
| **No `manifest_*` calls outside install.sh today** | Other entrypoints (doctor, update, uninstall) read the files via `jq` directly. `manifest_*` helpers are write-only from outside install.sh, by design — fewer entrypoints into mutation. |
