# Task 3.4.1 — Repo skeleton

**Status:** ✅ DONE 2026-04-28
**Estimated effort:** ~30 min
**Depends on:** —
**Blocks:** 3.4.2, 3.4.3, 3.4.4

## Goal

Create the empty directory structure that everything else in Phase 3
lands inside. Delete two unused legacy files. After this task, the
repo *looks* like the architecture doc says it should, even if most
folders are still empty.

## Why

`docs/architecture.md §3a` describes a target repo layout. Today
several pieces don't exist (`lib/engines/`, `src/`, `tests/`). Making
the empty scaffolding *first* lets all subsequent tasks drop files
into the right places without fighting structure decisions.

## Scope

### Files to create

```
lib/engines/README.md         # explains what belongs here
src/                           # TypeScript root
├── tsconfig.json              # bun-compatible, strict mode
├── package.json               # name=claudify-src, type=module
└── lib/                       # placeholder for future TS modules
    └── README.md
tests/                         # tests root
├── README.md                  # explains the two suites
├── bash/
│   ├── README.md
│   └── canary.bats            # one bats test that always passes
└── ts/
    ├── README.md
    └── canary.test.ts         # one bun test that always passes
test.sh                        # repo-root entry: runs both suites
```

### Files to modify

- `.gitignore` — add `node_modules/`, `src/dist/`, `bun.lockb`
- `README.md` — brief mention under "Development" that `bash test.sh` exists
- `lib/README.md` — add row explaining `lib/engines/`

### Files to delete

- `templates/access.json`        — unused (heredocs are inline in `lib/steps.sh`)
- `templates/claude-telegram.service` — same
- `templates/README.md`           — folder becomes empty; remove the README too

## Steps

1. `mkdir -p lib/engines src/lib tests/bash tests/ts`
2. Write each README.md stub explaining what belongs in that folder
3. Write `src/tsconfig.json` (strict, target ES2022, module ESM, types from bun)
4. Write `src/package.json` minimal — `name`, `type: module`, `dependencies: {}`
5. Write `tests/bash/canary.bats` — one test that asserts `1 == 1`
6. Write `tests/ts/canary.test.ts` — one `bun test` that asserts the same
7. Write `test.sh` at repo root: detects bats + bun, runs both suites, exits non-zero if either fails or is missing
8. `git rm` the three `templates/` files
9. Update `.gitignore` (`node_modules/`, `bun.lockb`, `src/dist/`)
10. Update `lib/README.md` function table with `lib/engines/` row
11. `bash -n test.sh` + `bash test.sh` (locally — bats may not be installed; should warn-skip not fail)
12. `bash build.sh` — confirm dist still builds
13. CHANGELOG entry in Unreleased: `### Added — repo skeleton: lib/engines/, src/, tests/. Removed unused templates/.`
