## What landed

Ctrl-C anywhere in `install.sh` after pasting your three inputs, and
re-running picks up exactly where you stopped. No `--resume` flag, no
"Would you like to resume?" prompt, no env var to remember. Just
works. The whole point: keep the install ridiculously easy.

## Design choices

| Decision | Why |
|---|---|
| **No `--resume` flag, no prompt** | Owner explicitly asked for this: *"if it's become heavy so I miss the all point of all of that."* The simplest UX is no UX — invisible until needed. |
| **File at `$CLAUDIFY_ROOT/.install-partial`** | Inside the directory `uninstall.sh` already wipes (`rm -rf ~/.claudify`), so cleanup on uninstall is free. Hidden filename signals "internal — don't edit." |
| **chmod 600 + `umask 077` on write** | Bot token is in there. Same belt-and-suspenders pattern as `credentials.env` and `telegram/.env`. |
| **Persisted in `_collect_inputs_fresh` only, not `_collect_inputs_preserved`** | `--preserve-state` already has `telegram/.env` as its source of truth — partial would be redundant. |
| **`set -a; . file; set +a` to source** | Auto-exports each variable. Plain `KEY=value` lines (no quoting) so unusual chars in tokens don't get mangled. Bot tokens are `[0-9]+:[A-Za-z0-9_-]+` — safe. |
| **Skip on `DRY_RUN=1`** | Dry-run is a preview; reading partial state during a preview would muddy "what *would* happen if I ran this." |
| **Skip on `PRESERVE_STATE=1`** | Update flow has its own canonical state at `telegram/.env`; partial file is for the initial install only. |
| **Env vars win over partial state** | Operator pre-setting `BOT_TOKEN=…` is intentional — that's the explicit override path. Partial is a fallback, not a default. |
| **Cleanup only on success path** | Failure mid-install means the operator wants to resume. Wiping the partial on failure would defeat the whole point. |

## Station11 round-trip — three scenarios

### 1. Partial-state write
```text
ssh ... 'rm -rf ~/.claudify'
ssh ... 'BOT_TOKEN=… TG_USER_ID=… WORKSPACE=… bash /tmp/install.sh --non-interactive &'
ssh ... 'wait for ~/.claudify/.install-partial; SIGINT installer'

→ -rw------- 1 david david 98 May  4 10:52 .install-partial
→ exactly 3 lines: BOT_TOKEN=… / TG_USER_ID=9999999 / WORKSPACE=claude-bot
```

### 2. Resume on re-run with no env vars
```text
ssh ... 'unset BOT_TOKEN TG_USER_ID WORKSPACE; bash /tmp/install.sh --non-interactive'

→ === Configuration ===
→ ✓ resumed from previous attempt — using saved BOT_TOKEN, TG_USER_ID, WORKSPACE
→ (zero "Paste your Telegram …" prompts in the log)
→ ✓ bot token written
→ ✓ allowlist written (user 9999999)
```

### 3. Cleanup on success
```text
ssh ... 'full scrub + pre-seed real OAuth + bash /tmp/install.sh --non-interactive'

→ exit code: 0
→ Claudify  —  install complete
→ ls ~/.claudify/.install-partial
→   No such file or directory   ✓
→ systemctl --user is-active claude-telegram.service → active
→ doctor → All 28 checks passed
```

Temp files (`/tmp/install-*.sh`, `/tmp/install-*.log`) shredded after.

## Notes for next tasks

Pre-existing from 3.4.2's split. Worth fixing in a one-line follow-up — not on the roadmap yet, low priority since real `--preserve-state` invocations always have an existing install.
