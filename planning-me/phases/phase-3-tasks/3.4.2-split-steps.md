# Task 3.4.2 — Split `lib/steps.sh` into focused modules

**Status:** ✅ DONE 2026-04-30
**Estimated effort:** ~1 hr
**Depends on:** 3.4.1
**Blocks:** 3.4.3 (engine extraction needs the module boundaries to exist)

## Goal

Break `lib/steps.sh` (currently ~430 lines, well over our 300-line
limit) into 5 focused modules. Each module owns one cohesive concern
and stays well under the limit. `install.sh` updates its source order;
`build.sh` updates its MODULES array.

## Why

`CLAUDE.md` rule 1: bash files ≤ 300 lines. `steps.sh` violates it.
`docs/architecture.md §3a` already lists the target file structure.
This is the prerequisite for the engine abstraction (3.4.3) — engine-
specific code can't be cleanly extracted from a 430-line catch-all.

## Scope

### Files to create (under `lib/`)

| New module | Functions moved into it (from `steps.sh`) | Approx LoC |
|---|---|---|
| `onboarding.sh` | `intro`, `guide_botfather`, `guide_userinfobot`, `collect_inputs` | ~110 |
| `claude.sh` (TEMP — gets renamed in 3.4.3) | `setup_npm_prefix`, `install_claude`, `install_telegram_plugin`, `seed_claude_state`, `claude_is_authed` | ~140 |
| `configs.sh` | `write_configs`, `seed_persona` | ~100 |
| `service.sh` | `write_service`, `start_service`, `final_summary` | ~100 |
| `oauth.sh` | `oauth_setup` (+ helpers if any) | ~70 |

> Note: `claude.sh` is a temporary holding name. In 3.4.3 its contents
> get split between a real engine adapter (`lib/engines/claude-code.sh`)
> and engine-agnostic glue. Don't worry about that boundary now —
> just get everything out of `steps.sh`.

### Files to modify

- `install.sh` — update `source` order in the orchestrator
- `build.sh` — update the `MODULES` array
- `lib/README.md` — function table listing all modules and what they expose

### Files to delete

- `lib/steps.sh` — gone after the split

## Steps

1. Read `lib/steps.sh` end-to-end and identify the boundaries (already mapped above)
2. Create the 5 new files; copy each function block + its preceding comment into the right one
3. Move shared constants (`NPM_PREFIX`, `CLAUDIFY_ROOT`, `CLAUDIFY_WORKSPACE`, `CLAUDIFY_TELEGRAM`, `CREDS_FILE`) to a sensible home — likely top of `claude.sh` for now (engine-specific paths) and `lib/ui.sh` for `LOG_FILE`-style ones if any. Don't duplicate.
4. Update `install.sh` source order: `ui → args → prompts → validate → preflight → onboarding → claude → configs → service → oauth → main()`
5. Update `build.sh` MODULES array to match
6. Update `lib/README.md` function table
7. Delete `lib/steps.sh`
8. `bash -n` every modified/created file
9. `for f in lib/*.sh; do wc -l "$f"; done` — confirm every file ≤ 300 lines
10. `bash build.sh` — `dist/install.sh` rebuilds clean
11. **Round-trip test on Station11:** `bash ~/test/update.sh` — must complete green
12. CHANGELOG entry: `### Changed — lib/steps.sh split into 5 focused modules (onboarding, claude, configs, service, oauth) to comply with the 300-line file limit.`
